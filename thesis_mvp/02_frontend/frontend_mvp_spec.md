# 前端MVP规范（混合模式）

## 产品形态
- 客户门户（外部）：提交需求、查看状态、下载交付、提交反馈
- 内部控制台（运营）：查看节点轨迹、置信度、案例命中、人工介入记录

## 页面最小清单

### 1) 客户门户 - 新建需求页
- 输入方式：
  - 表单录入（单条）
  - Excel 上传（批量）
- 字段校验：
  - 必填：`category`、`item_name`、`quantity`、`required_date`
  - 风险提醒：`spec` 为空时给出“需补充关键规格”的黄色提示
- 提交后动作：
  - 创建 run（状态：`queued`）
  - 返回任务编号 `run_id`

### 2) 客户门户 - 任务列表/详情页
- 列表展示：
  - run_id、品类、状态、创建时间、最新更新时间
- 详情展示：
  - 当前节点（supervisor/analysis/research/recommendation）
  - 预计完成状态
  - 下载按钮：PO 草案、节省报告 PDF

### 3) 客户门户 - 反馈页
- 反馈字段：
  - 是否采纳（是/否/部分采纳）
  - 不采纳原因（可选）
  - 评分（1-5）
  - 备注（可选）

### 4) 内部控制台 - 轨迹页
- 时间线：
  - 每个节点输出 `main_output`
  - `reasoning` 与 `confidence`
- 审计信息：
  - 路由决策来源（LLM / rule-fallback）
  - 是否触发循环安全阀

### 5) 内部控制台 - 运营看板页
- KPI 卡片：
  - 平均处理时长
  - 人工介入率
  - 建议采纳率
  - 估算节省率
- 分组维度：
  - 品类（MRO/包装）
  - 周/月时间窗

## 演示脚本（答辩 8-10 分钟）
1. 在客户门户提交 1 条 MRO 需求。
2. 在任务详情页看到状态从 queued 到 completed。
3. 打开内部控制台，展示节点轨迹与置信度变化。
4. 下载节省报告 PDF，讲解 benchmark 与推荐理由。
5. 回到反馈页提交“部分采纳”，展示数据如何沉淀到反馈表。

## 技术实现建议（最小可行）
- 前端框架：Streamlit 或轻量 React 均可。
- 本科答辩优先建议：先用 Streamlit 快速成型（1 周内可完成）。
- 页面风格：功能优先，不做重视觉投入。
